# This is NOT the Next.js you know

This version has breaking changes — APIs, conventions, and file structure may all differ from your training data. Read the relevant guide in `node_modules/next/dist/docs/` before writing any code. Heed deprecation notices.
<!-- END:nextjs-agent-rules -->

## Critical commands

```bash
pnpm lint -> pnpm typecheck -> pnpm test   # verify before committing
pnpm api:refresh                           # regenerate API client after backend OpenAPI change
pnpm api:diff                              # compare live backend /docs-json vs committed openapi.json (CI pin)
```

## Never edit generated code

`src/lib/api/generated/` is generated by Orval from `backend/openapi.json`. Never modify these files — run `pnpm api:refresh` when the backend spec changes.

## API / cookie coordination

- Backend runs on `http://localhost:3000` (cookie domain `localhost`)
- E2E tests run the Next.js dev server on `localhost:3001` (`NEXT_PUBLIC_API_URL=http://localhost:3000` so cookies still work)
- Edit `.env.local` not `.env.example`

## Next.js 16 / React 19 async APIs

`cookies()`, `headers()`, `params` in Server Components and Route Handlers are now `async`. Read `node_modules/next/dist/docs/` rather than relying on older Next.js knowledge.

## Role enum is UPPERCASE

Backend enum is `ADMIN` / `DOCTOR` / `PATIENT` (not lowercase). The generated schemas use the uppercase form.

## Test environments

- Vitest: `node` environment by default. Add `/** @vitest-environment jsdom */` to component test files that need DOM.
- Playwright E2E: `workers: 2` max (cold-compilation parallelism limit). Runs on `http://localhost:3001`.
